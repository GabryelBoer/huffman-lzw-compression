## Directory Purposes

**`.planning/`:**
- Purpose: Orchestrator-managed planning documents (ARCHITECTURE.md, STRUCTURE.md, etc.)
- Contains: Markdown analysis files generated by GSD tools
- Key files: `codebase/ARCHITECTURE.md`, `codebase/STRUCTURE.md`

**`src/huffman/`:**
- Purpose: Huffman coding algorithm implementation
- Contains: Encoder, decoder, tree construction via heap
- Key files: `encoder.py` (main interface), `tree.py` (algorithm core)

**`src/lzw/`:**
- Purpose: LZW dictionary-based compression algorithm
- Contains: Encoder, decoder with adaptive dictionary (256-4096 codes)
- Key files: `encoder.py`, `decoder.py` (both encode and decode in one file per algorithm)

**`src/io/`:**
- Purpose: Infrastructure for sub-byte I/O operations
- Contains: BitWriter (accumulate bits, flush to bytes), BitReader (traverse padded bitstream)
- Key files: `bitstream.py` (single file, 75 lines)

**`src/benchmark/`:**
- Purpose: Orchestration, analysis, and metrics for compression benchmarking
- Contains: CLI entry point, performance measurement, file characterization
- Key files:
  - `compare.py`: Discovers files, runs both algorithms, writes CSVs
  - `metrics.py`: Timing, memory profiling, SHA-256 validation
  - `profile.py`: Shannon entropy, lexical token analysis, file categorization

**`data/`:**
- Purpose: Input test files for benchmarking
- Contains:
  - `baseline/access.log`: Repetitive log for algorithm comparison baseline
  - `codigo/py/`: Python source samples (small, medium, high-entropy)
  - `codigo/c/`: C source samples for language-specific compression testing

**`results/`:**
- Purpose: Benchmark output (generated by `compare.py`)
- Contains: CSV files (summary and per-file detail)
- Key files: `compression-summary.csv` (all results), `compression-detail-*.csv` (per-file)

**`benchmark-plots/`:**
- Purpose: Visualization output (generated by `plot-results.py`)
- Contains: PNG charts (compression ratio, time, memory)
- Key files: `compression-ratio.png`, `tempo-execucao.png`, `memoria.png`

**`tests/`:**
- Purpose: Automated test suite (pytest)
- Contains: Round-trip validation (compress → decompress == original)
- Key files: `test_roundtrip.py` (parametrized tests for both algorithms)

## Key File Locations

**Entry Points:**
- `src/benchmark/compare.py`: Main CLI entry (run via `python -m src.benchmark.compare` or from `run-all.sh`)
- `plot-results.py`: Visualization entry (standalone script)
- `tests/test_roundtrip.py`: Test entry (pytest collected)

**Configuration:**
- `requirements.txt`: Python dependencies (matplotlib, numpy, pytest)
- `.gitignore`: Git exclusion rules
- `run-all.sh`: Orchestration script (sets PYTHONPATH, runs benchmark + visualization)

**Core Logic:**
- Huffman: `src/huffman/encoder.py`, `src/huffman/decoder.py`, `src/huffman/tree.py`
- LZW: `src/lzw/encoder.py`, `src/lzw/decoder.py`
- Analysis: `src/benchmark/compare.py`, `src/benchmark/metrics.py`, `src/benchmark/profile.py`
- I/O: `src/io/bitstream.py`

**Testing:**
- `tests/test_roundtrip.py`: Roundtrip validation for both codecs

## Naming Conventions

**Files:**
- Module files (codec entry points): `encoder.py`, `decoder.py`, `tree.py`
- Analysis modules: `metrics.py`, `profile.py`, `compare.py`
- Infrastructure: `bitstream.py`
- Scripts: lowercase with hyphens for executables (`run-all.sh`, `plot-results.py`)
- Tests: `test_*.py` (pytest convention)

**Directories:**
- Algorithm packages: lowercase algorithm name (`huffman/`, `lzw/`)
- Feature modules: lowercase domain name (`io/`, `benchmark/`)
- Top-level artifacts: lowercase domain with optional hierarchy (`data/`, `results/`, `tests/`)

**Python Classes:**
- PascalCase: `HuffmanNode`, `BitWriter`, `BitReader`, `FileProfile`, `CompressionResult`

**Python Functions:**
- snake_case: `encode()`, `decode()`, `build_frequency_table()`, `analyze_file()`, `measure_compression()`

## Where to Add New Code

**New Compression Algorithm:**
- Create new package: `src/{algorithm_name}/` with `__init__.py`
- Implement `encoder.py` and `decoder.py` with signatures:
  - `def encode(data: bytes) -> bytes:` — must write magic bytes and format header
  - `def decode(data: bytes) -> bytes:` — must validate magic bytes
- Update `src/benchmark/compare.py:run_benchmark()` to add tuple to algorithm loop (line 74-77)

**New Benchmark Metric:**
- Add field to `CompressionResult` dataclass (`src/benchmark/metrics.py`)
- Implement measurement logic in `measure_compression()` function
- Update CSV column mapping in `result_to_row()` (`src/benchmark/compare.py:42-58`)

**New File Analysis Feature:**
- Add method to `FileProfile` dataclass or create new analysis function in `src/benchmark/profile.py`
- Call from `analyze_file()` at line 81
- Add corresponding CSV column in `result_to_row()` if needed for output

**New Visualization:**
- Add function to `plot-results.py` following pattern of existing plots (lines 20-77)
- Call from `main()` function before final print
- Load required columns from CSV in load_summary()

**New Test:**
- Add to `tests/test_roundtrip.py` as parametrized or standalone pytest function
- Follow naming convention `test_{feature}_{condition}()`

**New Utility/Infrastructure:**
- If bit-level: extend `src/io/bitstream.py`
- If general: add to appropriate package or create new `src/utils/` if cross-cutting

## Special Directories

**`.venv/`:**
- Purpose: Python virtual environment
- Generated: Yes (created by `python -m venv`)
- Committed: Yes (full environment tracked in git, not using .gitignore)

**`.git/`, `.pytest_cache/`:**
- Purpose: Git repository metadata, pytest cache
- Generated: Yes
- Committed: Only `.git/` (cache excluded via .gitignore)

**`benchmark-plots/`, `results/`:**
- Purpose: Generated benchmark artifacts
- Generated: Yes (by `compare.py` and `plot-results.py`)
- Committed: Yes (baseline results tracked for comparison)
