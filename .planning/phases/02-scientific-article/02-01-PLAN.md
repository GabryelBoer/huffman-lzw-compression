### **Task 5: Results & Analysis (Day 3–4)**
**Goal:** Verify Results section matches data from Phase 1 benchmarks and interpret findings (DOC-07)  
**Owner:** Team  
**Effort:** 2–3 hours

**Steps:**
1. Open `artigo.tex` and locate Results section
2. Verify **3 figures present and properly referenced:**
   - Figure 1: Compression Ratio (%) by file type
   - Figure 2: Execution Time (ms)
   - Figure 3: Memory Usage (MB)
   - Each must have: caption above/below, figure reference in text, explanation in prose
3. Verify **3 tables present:**
   - Table 1: Dataset summary (file types, sizes, entropy)
   - Table 2: Results summary (compression ratio, time, memory)
   - Table 3: Comparison by file type (which algorithm wins in each category)
4. **Critical: Interpret results, don't just present data**
   - For each result: "This shows that [algorithm] is better at [condition] because [mechanism]"
   - Example: "Huffman achieved 32% compression on Python files due to the distribution of whitespace and tokens"
   - Example: "LZW expanded C headers by 4% because inline patterns are too short for dictionary utilization"
5. Verify **empirical-to-theory connection:**
   - Does O(n log n) Huffman align with observed time on larger files?
   - Does O(n) LZW align with observed time across all categories?
   - Note any deviations and explain them (GC overhead, memory cache effects, etc.)
6. Check for **neutral language:**
   - Avoid "obviously better" — say "wins at metric X in category Y"
   - Include cases where both algorithms tie or LZW wins
7. Update audit with results completeness

**Success Criteria:**
- All 3 figures present with captions and references
- All 3 tables present with captions and references
- Each result is interpreted (1–2 sentences per table/figure)
- Theory-practice connection is explicit
- No unsupported claims
- Page range: 1.75–2.25 pages

### **Task 7: Portuguese Proofread (Day 5–6)**
**Goal:** Ensure Portuguese norma culta throughout (DOC-09)  
**Owner:** Team (native Portuguese preferred)  
**Effort:** 3–4 hours

**Steps:**
1. Read entire PDF (compiled in Task 1) from start to finish
2. Check for **common Portuguese errors:**
   - Verb agreement (subject-verb number/person match)
   - Preposition choice (common substitutions: "em" vs "a", "de" vs "para")
   - Pronoun reference (ambiguous antecedents)
   - Tense consistency (avoid mixing past/present without reason)
   - Accent marks (é, à, ão, ç) — absent in many environments
3. Check for **academic tone issues:**
   - Avoid first-person singular ("eu acredito") → use passive or collective ("acredita-se", "nossos resultados")
   - Avoid colloquialisms ("coisa", "tipo de", "meio que")
   - Use formal transitions ("Para isto", "Portanto", "Em resumo")
4. Check for **clarity and "naturalidade":**
   - Read sentences aloud; they should sound natural, not translated
   - Long sentences should be broken (max 20 words per sentence for academic clarity)
   - Pararaph structure: topic sentence first, then development
5. Create list of errors found (in `02-AUDIT.md`)
6. For each error: mark line/section, describe issue, suggest correction
7. Focus on **impact errors** first:
   - Meaning-changing (e.g., "não compressão" vs "compressão não")
   - Clarity-reducing (ambiguous pronouns, unclear references)
   - Tone-breaking (colloquialisms, first-person bias)

**Success Criteria:**
- No verb/preposition/agreement errors
- Academic tone consistent
- Portuguese reads naturally (not translated-sounding)
- Common errors log created (for context on correction effort)
